# 一期验证指南：昇腾算子远程实训平台

本指南定义一期端到端验收，不替代实现说明。字段、错误码和请求格式以
[控制面契约](contracts/phase1.openapi.yaml) 为准；实体和状态以
[数据模型](data-model.md) 为准。

## 前置条件

1. 受控 Linux 单节点环境已部署控制面、Worker、持久存储、终端采集和监控。
2. 统一身份提供者可发放稳定 subject；至少准备平台管理员、教师、助教、学生与查看者主体。
3. 准备两个班级、各自成员、CPU/NPU Profile、独立额度和至少一个健康 CPU 运行节点。
4. 如验证 NPU，准备通过硬件准入的 Orange Pi AIPro 8T 或 20T 节点、目标镜像与可用 device-share
   Adapter；每个节点预留至少 3 个 NPU 工作区容量。
5. 已配置受控持久卷、备份恢复演练目标、终端敏感信息测试样本和操作/审计查询权限。

## 启动与验证入口

实现仓库提供后，在根目录运行：

```text
make verify
```

该入口必须依次覆盖格式、静态检查、单元、数据库/Adapter 集成、OpenAPI 契约、端到端和架构检查。若
命令或环境尚未建立，不得把文档渲染成功当作平台功能验证。

## 验收场景

### 1. 身份、班级与可扩展 RBAC

1. 平台管理员创建/启用 Profile，并为两个班级分别启用 CPU/NPU 配置和额度。
2. 在班级 A 创建教师、助教、学生和查看者绑定；验证授权者可管理角色、权限和绑定，且每次变更有审计。
3. 使用班级 B 主体访问班级 A 的成员、工作区、日志、任务和审计；预期均被拒绝且不泄露存在性。

### 2. CPU 工作区闭环

1. 班级 A 学生选择批准且启用的 CPU Profile 创建工作区，记录 `operationId`。
2. 连续执行至少 20 次创建；预期至少 19 次在 3 分钟内到达可用或明确失败终态。
3. 对一个运行中工作区执行停止、启动和默认重置；预期持久数据保留，重置成功后进入运行中。
4. 删除时分别选择保留数据和同时删除数据；验证卷归属、配额影响、重新绑定/自动删除重试和审计结果。

### 3. NPU 工作区与准入

1. 班级 A 学生选择批准且启用的 NPU Profile 创建工作区；验证 NPU 配额和节点槽位已预留。
2. 在同一健康目标节点并发运行 3 个受控 NPU 工作区；验证工作负载相互隔离、统计准确且无降级为 CPU。
3. 尝试第 4 个工作区或超过编译队列限制的请求；预期操作进入可解释等待/失败状态。
4. 对 8T 和 20T（若均纳入试点）分别执行资源发现、创建、停止和删除验证，记录 Adapter 兼容证据。

### 4. 三种入口与终端证据

1. 同一学生通过 WebIDE、SSH 和 VS Code Remote-SSH 进入同一工作区，验证看到同一持久数据。
2. 分别产生输入、stdout、stderr 及预定义敏感内容样本；教师/助教仅能在班级 A 查询过滤后的实时和归档记录。
3. 模拟采集短暂中断；预期工作区仍可安全处理，运营视图存在关联缺口/恢复证据和告警。

### 5. 培训期结束与长期任务

1. 对班级 A 发起培训期结束；预期学生工作区停止、数据保留、学生新操作受限，教师/授权管理员可导出。
2. 仅使用授权管理员发起清理；验证每个数据最终状态可查询和审计。
3. 由授权主体提交长期任务并分配任务范围权限；验证产生独立 Task Workspace/Volume，未经任务授权的
   同班级主体及其他班级主体均无法访问。
4. 验证任务不能读取学生卷或学生终端记录，也不产生任何服务端点或公网暴露。

### 6. 恢复、备份与运营

1. 重放相同幂等键、重复 Worker 消息、模拟运行时超时和控制面中断；预期不产生重复资源，旧结果不覆盖新 generation。
2. 模拟“运行资源已删除、卷删除失败”；预期 Operation 保持执行/协调并自动重试，用户不能重复发起删除。
3. 对两个班级的控制面数据和持久卷进行备份恢复演练；验证恢复后归属、授权边界和操作记录仍正确。
4. 检查 `/healthz`、`/readyz`、Trace Context、结构化日志、审计、配额/操作/采集指标与告警；确认无令牌、
   Cookie、密码或私钥泄露。

### 7. 前端响应式、组件与预加载

1. 在 375px、768px、1024px 与 1440px 视口访问“我的工作区”、详情、教学过程和长期任务；预期页面
   均使用 [前端 UI 契约](contracts/frontend-ui.md) 的 12 栏 grid，窄屏通过跨栏收敛，宽表有摘要或受控滚动。
2. 延迟班级、工作区、终端和审计的独立读取；预期 route `loading` 和区块 Skeleton 先稳定显示、已完成
   区块可先呈现，且 Skeleton 不含名称、数量、状态或授权线索。
3. 用键盘完成导航、对话框开关和命令提交；预期焦点管理正确、重复提交被阻止、操作 ID 可见，状态不只
   依赖颜色或动画。
4. 以教师、授权管理员和学生分别调用 OpenAPI 1.3 的工作区操作历史/卷、终端筛选、培训期导出/清理、
   Task Grant、额度与运营摘要端点；预期授权范围、不可枚举响应、幂等 Operation、分页游标和安全字段均
   与 [控制面契约](contracts/phase1.openapi.yaml) 一致。
5. 以授权管理员完成权限定义 CRUD、角色/绑定管理、成员移除、平台 Profile 与班级 Profile 额度治理；
   预期每个变更均携带 `traceparent` 与（提供时）`tracestate`，且形成审计记录。删除工作区时省略
   `dataDisposition` 必须被契约拒绝；导出成功后仅可用 Operation 的受控结果下载，下载请求再次授权并审计。
